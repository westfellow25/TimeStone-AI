The math behind the numbers

Methodology

Every number TimeStone reports has a derivation. Nothing is sleight of hand.

$ Financial model (NPV)

Capex paid in year 0. Benefits begin after the implementation period. 5-year NPV discounted at WACC (default 12%, configurable).

NPV = -Capex + Σ y=1..5 ( benefit_y · adoption_y ) / (1 + 0.12)y

Adoption ramps year by year, not full impact from day one:

! Risk model

Each Monte Carlo iteration draws revenue / cost impact from Gaussian distributions scaled by risk level. Then independent shocks fire:

  • Execution failure (per-scenario)2 — 60%
  • Market downturn8%
  • Competitive response15%
  • Cost overrun+10 — 50%
  • Implementation delay+20 — 80%

Per-scenario execution_failure_prob is derived from the empirical failure rate of similar real cases — not picked by hand.

★ Empirical priors

Instead of asking "what is a reasonable revenue uplift for digital booking?", TimeStone asks "what did Maersk Spot, Hertz/Accenture, Halyk Homebank actually deliver?"

prior_mean = w · empirical_mean + (1 − w) · template_mid
where w = min(n_cases / 5, 1.0)

At n≥5 similar cases, the empirical mean is used fully. At lower n, it shrinks toward a conservative template midpoint.

∞ Bayesian calibration loop

Every real engagement produces an OutcomeRecord — append-only, never deleted. The system reads these to compute per-bucket residuals (actual minus predicted) and shifts future priors automatically.

residual = actual − predicted
shift = residual · min(n / 5, 1.0)
prior_mean_next = prior_mean + shift

More engagements → tighter calibration → harder for any competitor to catch up. This is the compounding moat.

Get started

Install & run on your data

Five seed companies bundled. One command produces a full assessment with executive PDF. 

# Clone & install
git clone https://github.com/westfellow25/timestone-ai
cd timestone-ai
pip install -e .

# List bundled twins
python -m timestone list-companies

# Run a full assessment (produces PDF + JSON artifacts)
python -m timestone assess "Kazakhstan Temir Zholy (KTZ)"

# Browse past runs and re-generate any PDF
python -m timestone list-runs
python -m timestone report <run_id>

# Process outcomes and recompute calibration
python -m timestone calibrate

# Or launch the interactive dashboard
streamlit run src/timestone/interfaces/web/dashboard.py
